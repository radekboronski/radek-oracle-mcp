# Radek Oracle MCP

**Advanced Oracle 19c diagnostic MCP server** for AI tools (Claude Code, Cursor).

**29 tools** covering the full Oracle diagnostic stack: sessions, locks, wait events, SQL performance (v$sql), tablespaces, SGA/PGA memory, redo logs, undo management, optimizer statistics, Data Guard, and schema analysis.

## Quick Start

### 1. Install dependencies

```bash
cd /data/project/radek-oracle-mcp
npm install
npm run build
```

### 2. Initialize config and generate secret

```bash
cp db.config.example.json db.config.json
node dist/create-token.js --init
```

### 3. Generate a JWT token

```bash
node dist/create-token.js --name claude --days 365
```

Output includes a ready-to-use `.mcp.json` block for Claude Code and Cursor.

### 4. Start the server

```bash
npm start
# or for development:
npm run dev
```

Verify:
```bash
curl http://localhost:8003/health
# {"status":"ok","server":"Radek Oracle MCP","version":"1.0.0","tools":29}
```

### 5. Start the Cloudflare Tunnel

```bash
cloudflared tunnel --config ~/.cloudflared/config-radek-oracle-mcp.yml run radek-oracle-mcp
```

## Configuration

**db.config.json:**

```json
{
  "auth": {
    "secret": "<64-char hex — generated by node dist/create-token.js --init>"
  },
  "server": {
    "port": 8003,
    "tunnel_url": "https://radek-oracle-mcp.clickchain.eu"
  }
}
```

> `database.url` is optional — Claude passes the Oracle URL dynamically via `X-Database-URL` header.

## Oracle Connection URL Format

```
X-Database-URL: oracle://user:password@host:1521/service_name
X-Database-URL: oracle://user:password@host:1521/ORCL
X-Database-URL: oracle://user:password@scan-host:1521/MYDB    (RAC)
X-Database-URL: oracle+ssl://user:password@host:2484/ORCL     (SSL/TLS)
```

### Required Oracle Privileges

```sql
GRANT CREATE SESSION TO mcp_user;
GRANT SELECT_CATALOG_ROLE TO mcp_user;
GRANT SELECT ANY DICTIONARY TO mcp_user;
-- For explain_query:
GRANT CREATE ANY TABLE TO mcp_user;
-- Or simply:
GRANT DBA TO mcp_user;
```

## MCP Client Configuration

### Claude Code — `.mcp.json`

```json
{
  "mcpServers": {
    "radek-oracle": {
      "type": "http",
      "url": "https://radek-oracle-mcp.clickchain.eu/mcp",
      "headers": {
        "Authorization": "Bearer <your-token>",
        "X-Database-URL": "oracle://user:password@host:1521/ORCL"
      }
    }
  }
}
```

### Claude Code — CLI

```bash
TOKEN="<your-token>"
DB="oracle://user:password@host:1521/ORCL"

claude mcp add radek-oracle https://radek-oracle-mcp.clickchain.eu/mcp \
  --transport http \
  --header "Authorization: Bearer $TOKEN" \
  --header "X-Database-URL: $DB" \
  --scope local
```

## Cloudflare Tunnel

The tunnel `radek-oracle-mcp` was created with ID `ffcf0846-8f91-4687-a312-989ac2e11835`.

DNS `radek-oracle-mcp.clickchain.eu` → points to this tunnel.

### Start manually

```bash
cloudflared tunnel --config ~/.cloudflared/config-radek-oracle-mcp.yml run radek-oracle-mcp
```

### Config file (`~/.cloudflared/config-radek-oracle-mcp.yml`)

```yaml
credentials-file: /home/mcp/.cloudflared/ffcf0846-8f91-4687-a312-989ac2e11835.json
ingress:
  - hostname: radek-oracle-mcp.clickchain.eu
    service: http://127.0.0.1:8003
  - service: http_status:404
```

### systemd service

```bash
sudo tee /etc/systemd/system/cloudflared-radek-oracle-mcp.service > /dev/null << 'EOF'
[Unit]
Description=Cloudflare Tunnel — radek-oracle-mcp
After=network.target

[Service]
User=mcp
ExecStart=/usr/bin/cloudflared tunnel --config /home/mcp/.cloudflared/config-radek-oracle-mcp.yml run radek-oracle-mcp
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable cloudflared-radek-oracle-mcp
sudo systemctl start cloudflared-radek-oracle-mcp
```

## Available Tools (29)

| Category | Tool | Description |
|----------|------|-------------|
| **Diagnostics** | `auto_diagnose` | Full automatic Oracle diagnosis — always start here |
| **Diagnostics** | `health_check` | Quick health: version, SGA, buffer cache, sessions, waits |
| **Sessions** | `analyze_sessions` | Sessions by user/machine/program, long-running, long ops, limits |
| **Locks** | `analyze_locks` | Blocking sessions with kill commands, lock waits, transactions, DDL locks |
| **SQL Performance** | `analyze_sql` | Top SQL by elapsed/CPU/disk/buffer/executions from v$sql |
| **SQL Performance** | `explain_query` | EXPLAIN PLAN FOR + DBMS_XPLAN.DISPLAY(ALL) |
| **Wait Events** | `analyze_waits` | v$system_event, current waits, enqueues, latches, ASH |
| **Memory** | `analyze_memory` | SGA/PGA components, memory advisors, sort overflow |
| **Tablespaces** | `analyze_tablespaces` | Usage %, datafiles, free space, temp, autoextend risk |
| **Tables** | `analyze_tables` | Segment sizes, statistics, stale stats, partitions |
| **Tables** | `investigate_table` | Deep dive on one table (size, stats, indexes, constraints) |
| **Tables** | `list_tables` | All user tables with sizes sorted by size |
| **Indexes** | `analyze_indexes` | All indexes, unusable, invisible, low cardinality, duplicates |
| **Indexes** | `indexes_for_table` | All indexes for a specific table |
| **Indexes** | `suggest_index` | Generate CREATE INDEX (does NOT execute) |
| **Storage** | `analyze_storage` | Segments by schema, datafile I/O, extent fragmentation |
| **Redo** | `analyze_redo` | Log groups, switch frequency, archivelog status |
| **Undo** | `analyze_undo` | ORA-01555 risk, undo stats, undo segments |
| **Statistics** | `analyze_statistics` | Stale/missing optimizer stats, gather job status |
| **Data Guard** | `analyze_dataguard` | Apply/transport lag, archive dests, DB role |
| **Configuration** | `analyze_configuration` | Key parameters, non-default, security settings |
| **Schema** | `analyze_schema` | Invalid objects, missing PKs, FK without index, disabled constraints |
| **Schema** | `table_structure` | Columns, constraints, indexes for one table |
| **Actions** | `run_query` | Execute any Oracle SQL |
| **Actions** | `kill_session` | ALTER SYSTEM KILL SESSION 'sid,serial#' |
| **Reports** | `generate_report` | PDF / HTML / DOCX report from markdown |
| **Discovery** | `query_tools` | List all tools with parameters |
| **Discovery** | `query_queries` | Example natural-language questions |
| **Discovery** | `get_instructions` | Return CLAUDE.md instructions |

## Architecture

```
AI Client (Claude Code, Cursor)
    │  HTTPS + JWT Bearer + X-Database-URL header
    ▼
Cloudflare Tunnel (radek-oracle-mcp.clickchain.eu)
    │  ffcf0846-8f91-4687-a312-989ac2e11835
    ▼
HTTP Server (Node.js, port 8003)
    │  JWT verify → JSON-RPC 2.0
    ▼
MCP Handler (protocol 2025-11-25)
    │  tools/call dispatch
    ▼
Tool Layer (29 tools in src/tools/)
    │  Oracle SQL queries
    ▼
Query Layer (Oracle SQL in src/queries/)
    │  oracledb pool (max 5 connections, thin mode)
    ▼
Oracle 12c / 18c / 19c / 21c
```
